## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI → DreamerV3 training | User-controlled input (task, steps, config) drives subprocess behavior |
| Checkpoint loading | Checkpoint files from disk loaded into JAX subprocess - could be corrupted |
| Config YAML | User-provided YAML could contain malicious content if not validated |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-24-06 | Tampering | Checkpoint files | mitigate | Validate checkpoint format on load; Pydantic DreamerConfig validates all inputs |
| T-24-07 | Denial of Service | Long-running training | mitigate | Ctrl+C handling saves checkpoint; max steps limit; eval_only mode for quick verification |
| T-24-08 | Information Disclosure | Training metrics | accept | Metrics contain only loss/reward values, no sensitive data |
| T-24-09 | Elevation of Privilege | Config YAML loading | mitigate | YAML loaded via Pydantic model (no arbitrary code execution); safe_load used |

</threat_model>
