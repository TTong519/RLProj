Create DreamerSubprocess class that:
1. Spawns a Python subprocess via multiprocessing with XLA_PYTHON_CLIENT_MEM_FRACTION=0.4 set in subprocess environment (per D-04, D-05)
2. Communicates via stdin/stdout using JSON line-delimited protocol (per D-04)
3. Message types: CONFIG (subprocess → dreamerv3 config), TRAIN (start training), EVAL (run evaluation), CHECKPOINT (save/load), METRICS (streamed metrics), SHUTDOWN
4. Main process NEVER imports JAX/dreamerv3 - all imports happen inside subprocess (per D-06)
5. Subprocess sets up logging to stderr, streams JSON metrics to stdout
6. Graceful shutdown on SIGTERM - subprocess handles cleanup
7. Error handling: subprocess crashes are caught, main process receives error JSON
8. Timeout handling for stuck training

Key methods:
- __init__(config: DreamerConfig) - stores config, doesn't spawn yet
- spawn() - starts subprocess
- send_config(config_dict) - sends CONFIG message
- train(total_steps: int, eval_every: int) - sends TRAIN, yields METRICS
- evaluate(checkpoint_path: str, n_episodes: int) - sends EVAL, returns metrics
- save_checkpoint(path: str) - sends CHECKPOINT
- load_checkpoint(path: str) - sends CHECKPOINT
- shutdown() - sends SHUTDOWN, waits for process exit

1. Constructor: __init__(env: SurgicalEnv, obs_type: Literal["pixels", "state"])
2. Observation space: flat dict with:
   - For pixels (per D-10): "image" (H, W, 4) RGBA uint8 normalized to [0,1] float32 from render('rgb_array') at DreamerConfig.pixel_resolution (default 64x64), plus "depth" channel where supported
   - For state (per D-11): concatenated flat array (~50-100 dims) containing: qpos, qvel, gripper state (aperture, force), task target position, tissue deformation metrics (max displacement, volume change, contact forces)
   - Both modes include boolean flags: "is_first", "is_last", "is_terminal"
3. Action space: embodied action dict with same continuous array as SurgicalEnv action, plus optional "reset" boolean key
4. reset() → returns first observation with is_first=True
5. step(action_dict) → processes action, calls env.step(), returns observation with is_last/is_terminal from terminated/truncated
6. Handles the reset-in-action protocol: if action_dict.get('reset', False) is True, calls env.reset() and returns first observation
6. Properly converts numpy dtypes to embodied-compatible types
7. close() delegates to env.close()

Obs type "pixels" extraction:
- env.render() returns (H,W,3) RGB, convert to RGBA by adding alpha=255
- Resize to pixel_resolution if needed
- Normalize to [0,1] float32
- If simulator supports depth (MuJoCo/PyBullet render('depth_array')), include as 4th channel or separate "depth" key

Obs type "state" extraction:
- Get qpos, qvel from env._simulator.get_state() or Observation dataclass
- Get gripper state from simulator or env config
- Get task target from env._target_pos
- Get tissue metrics from scene/environment controller
- Concatenate in fixed order with normalization

| Subprocess GPU memory | JAX subprocess allocated 40% GPU memory - must not exceed or crash main process |
| File system checkpoints | Checkpoint files written by subprocess - could be corrupted or malicious |

| T-24-03 | Information Disclosure | Checkpoint files | accept | Checkpoints contain model weights only, no sensitive data; local filesystem |
