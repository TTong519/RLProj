| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-29-05 | T (Tampering) | apply_difficulty on reward instances | accept | Per-instance mutation; not persistent; reward instances are constructed fresh per env reset |
| T-29-06 | I (Information Disclosure) | DifficultyLevel values | accept | Public scalar values (0.0/0.5/1.0); no secrets |
| T-29-07 | E (Elevation of Privilege) | apply_difficulty bypass | mitigate | Method is internal to reward class; no external API surface that would allow arbitrary field mutation outside the mapped PARAM_BOUNDS keys |

| T-29-09 | T (Tampering) | CurriculumStageConfig.difficulty union | mitigate | Existing float users (scheduler sampling) are unaffected; enum users go through TaskRewardRouter normalization which validates the type |
