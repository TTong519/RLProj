## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CI runner egress -> PyPI / kubernetes-sigs GitHub releases | pytest-kind downloads kind 0.17.0 + kubectl binaries to ./.pytest-kind/{cluster-name}/ on first run; crosses the trust boundary between the CI runner and upstream binary hosts |
| kind cluster -> e2e Job pod | The e2e Job runs inside the ephemeral kind cluster and mounts the surg-rl-checkpoints PVC; no host path mounts, no privileged containers |
| e2e overlay -> k8s/base/secret.yaml | The overlay MUST NOT pull k8s/base/secret.yaml into the e2e path (the base kustomization references it; the e2e overlay bypasses the base) |

| T-39-02 | Information Disclosure | e2e Job mounting k8s/base/secret.yaml (registry/RLM credentials) | high | mitigate | e2e overlay references ONLY ../../base/pvc.yaml (NOT - ../../base, which would pull secret.yaml); e2e Job uses busybox:1.36 (no registry credentials needed); acceptance criterion greps the overlay does not contain `- ../../base` |
