```python
class DreamerSubprocess:
    def spawn(self) -> None:
        ctx = multiprocessing.get_context("spawn")
        parent_stdin, child_stdout = ctx.Pipe()    # <-- child_stdout here is the connection
        child_stdin, parent_stdout = ctx.Pipe()
        self._stdin = parent_stdin
        self._stdout = parent_stdout
        self.process = ctx.Process(
            target=_subprocess_main,
            args=(child_stdin, child_stdout, self.config),
            daemon=True,
        )
        self.process.start()
        child_stdout.close()
        child_stdin.close()
        # Wait for READY message via self._stdout.recv()
        ready_msg = self._read_message()
        if ready_msg.get("type") != "READY":
            raise RuntimeError(f"Subprocess failed to start: {ready_msg}")
```

       ```python
               if s == "\n":
       ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| subprocess.stdout → parent.recv() | JSON messages from JAX subprocess to DreamerSubprocess parent — untrusted channel (any JSON-decoded payload can carry arbitrary `type` / `error` / `metrics` fields). |
| filesystem → training_metrics.json | End-of-training metrics dump on disk — trusted file (written by training.py, read by benchmark aggregator). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-26-01 | T (Tampering) | `_subprocess_main` stdout channel | mitigate | `_JsonStdout` wrapper uses the Pipe connection's `send()` method (preserves parent process trust assumption); newline-stripped payload is opaque to the parent. No `os.fdopen` (which could be hijacked via FD injection). |
| T-26-02 | I (Information Disclosure) | `training_metrics.json` | accept | Local file on trusted filesystem; no PII; no cross-process exposure. |
| T-26-03 | S (Spoofing) | Plot color literal | accept | Cosmetic; affects only visualization, not data integrity. Constant value mismatch has no security impact. |
| T-26-04 | D (Denial of Service) | Subprocess `os.fdopen` failure | mitigate | The audit flagged the FD pattern as fragile on Windows. Replacing with `_JsonStdout` (which uses the Pipe connection API) removes the FD-leak / Windows-compat failure mode. |
| T-26-05 | E (Elevation of Privilege) | N/A | accept | No new privilege boundaries introduced. |

**Net risk:** low. All three patches are line-level fixes to non-security-critical code paths. The subprocess protocol change eliminates one fragile FD-wrapping pattern.
</threat_model>
