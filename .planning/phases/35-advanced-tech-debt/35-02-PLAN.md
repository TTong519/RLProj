<objective>
Close Phase 29 code-review finding WR-03 by eliminating the type lie in
`CurriculumStageConfig.difficulty` usage. The union `float | DifficultyLevel` is
already accepted, but it leaks an enum instance to consumers that expect a
scalar. This plan normalizes the value at the curriculum boundary and inside
the new `SurgicalEnv._setup_rewards()` helper so the reward builder always
receives a float.

Purpose: WR-03 noted that if a future plan wires `current_difficulty` into the
env's `_task_difficulty`, the env would receive a `DifficultyLevel` member and
fail or behave unexpectedly. Closing this now prevents a latent bug from
surfacing in v0.6.0 curriculum integration work.

Output: A corrected `current_difficulty` property, a refactored
`SurgicalEnv._setup_rewards()` method, and additional tests covering the enum
path.
</objective>
