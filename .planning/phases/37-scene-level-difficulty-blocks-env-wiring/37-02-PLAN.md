<objective>
Wire the override-precedence chain through `SurgicalEnv._setup_rewards` and extract `apply_params(params)` on the 6 task rewards so a scene with `difficulty_blocks` produces a hard-mode environment at construction. This is the env-wiring half of TASK-08 (SC#2 truth table + Pitfall 6 curriculum coexistence + TASK-09 additive gate).

Purpose: Plan 01 shipped the schema field; this plan makes the env actually consume it. The 4-level precedence `difficulty_blocks[level] > task.difficulty_level > config.difficulty > default 0.5` is the single documented resolution site. The rewards refactor (apply_params extraction) is a pure refactor that unlocks composed-dict application without changing the scalar-path output.

Output: `SurgicalEnvConfig.difficulty` field live; `_setup_rewards` additive blocks branch; `apply_params` on BaseRewardFunction + 6 task rewards; TestPrecedenceTruthTable + curriculum-coexistence tests green.

<decisions>
Adopted from 37-RESEARCH.md Open Questions (no CONTEXT.md exists):
- **Q2 — SurgicalEnvConfig.difficulty:** ADD `difficulty: float = 0.5` to `SurgicalEnvConfig` so SC#2's `config.difficulty > default 0.5` precedence level is real and truth-table-testable as 4 distinct levels (Pitfall 1 — currently the getattr is inert).
- **Q1 — apply_params mapping scope:** MINIMAL (Option a). Each reward's `apply_params(params)` maps ONLY the existing single `PARAM_BOUNDS` key → ctor field (per-reward mapping table in 37-PATTERNS.md:327-334). Document the INERT override surface in the truth-table test (most difficulty_blocks cells compose into the params dict but never reach a ctor field — Pitfall 2). Defer expansion (Option b) to a follow-up phase.
- **Q4 — curriculum + blocks coexistence:** Blocks apply ONLY when the resolved level is a `DifficultyLevel` enum (EASY/MEDIUM/HARD). Under `use_curriculum=True` with a continuous scalar (e.g. 0.37), blocks are INERT and the env falls back to `interpolate_params(scalar)`. The blocks branch is an ADDITIVE early-return grafted AFTER `self._task_difficulty = difficulty_float` and BEFORE the existing `if task_type is not None:` router branch — the existing continuous-curriculum path stays byte-identical (TASK-09 regression guard).
- **Q3 — field shape:** Consumed here as `dict[DifficultyLevel, DifficultyLevelConfig]` (shipped by Plan 01).
</decisions>
</objective>
