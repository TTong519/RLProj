**Full file content:**

```dockerfile
# syntax=docker/dockerfile:1
# Dockerfile for NVIDIA Jetson (arm64 + JetPack 6.0 L4T)
# JetPack 6.0 ships: Ubuntu 22.04, CUDA 12.4, cuDNN 9.x, TensorRT 10.x
# Build: docker buildx build --platform linux/arm64 -f Dockerfile.jetson -t surg-rl:jetson .
# Run:   docker run --runtime=nvidia surg-rl:jetson version --verbose

FROM --platform=linux/arm64 nvcr.io/nvidia/l4t-pytorch:r36.4.0-pth2.5.0

# Prevent interactive apt prompts
ENV DEBIAN_FRONTEND=noninteractive

# Install system dependencies for MuJoCo, PyBullet
# L4T base already has Python 3.10, pip, and CUDA libs
RUN apt-get update && apt-get install -y --no-install-recommends \
    libgl1 \
    libglu1-mesa \
    libglew2.2 \
    libx11-6 \
    libglib2.0-0 \
    && rm -rf /var/lib/apt/lists/*

WORKDIR /app

# Copy dependency definitions first for layer caching
COPY pyproject.toml pytest.ini ./

# Install surg-rl with core + tracking extras
# Skip [distributed] (Ray/RLlib untested on Jetson), skip [dev] (test deps not needed in runtime)
RUN pip install --no-cache-dir --upgrade pip \
    && pip install --no-cache-dir -e ".[tracking]"

# Copy source and assets
COPY src/ /app/src/
COPY scenes/ /app/scenes/
COPY configs/ /app/configs/

ENV PYTHONPATH=/app/src
ENV PYTHONUNBUFFERED=1

# CUDA is already on PATH and LD_LIBRARY_PATH from L4T base image
ENTRYPOINT ["python", "-m", "surg_rl.cli"]
CMD ["version", "--verbose"]
```

Key design decisions:
- `nvcr.io/nvidia/l4t-pytorch:r36.4.0-pth2.5.0` — JetPack 6.0 (L4T R36.4.0) with PyTorch 2.5.0 pre-compiled for aarch64. This is a public NGC container (no login required for pull).
- `--platform=linux/arm64` lock — Jetson is always arm64, never cross-compiled to amd64.
- Installs only `[tracking]` extra (core RL deps already satisfied). Excludes `[distributed]` (Ray untested on Jetson) and `[dev]` (test tooling not needed in runtime).
- System deps identical to CPU Dockerfile (libgl1, libglew2.2, etc.) — these are available via apt on L4T Ubuntu.
- No Python installation needed — L4T base already has Python 3.10+.
