  - pyproject.toml (dependency list, script entrypoint)
  - .dockerignore (exclusion patterns)
  - AGENTS.md (project conventions)
  </read_first>
  <action>
Restructure Dockerfile to single-stage for buildx cross-arch compatibility and add platform directives:

1. Replace the header line with:
   ```dockerfile
   # syntax=docker/dockerfile:1
   ```

2. Add platform ARGs after the syntax line:
   ```dockerfile
   ARG BUILDPLATFORM
   ARG TARGETPLATFORM
   ```

3. Replace `FROM python:3.11-slim as base` with:
   ```dockerfile
   FROM --platform=$BUILDPLATFORM python:3.11-slim
   ```

4. Keep system dependencies (same packages work on both amd64 and arm64):
   ```dockerfile
   RUN apt-get update && apt-get install -y --no-install-recommends \
       libgl1 \
       libglew2.2 \
       libglu1-mesa \
       libx11-6 \
       libglib2.0-0 \
       && rm -rf /var/lib/apt/lists/*
   ```

5. Set WORKDIR /app, then copy and install in a single stage:
   ```dockerfile
   WORKDIR /app
   COPY pyproject.toml pytest.ini ./
   COPY src/ src/
   COPY README.md README.md
   RUN pip install --no-cache-dir --upgrade pip \
       && pip install --no-cache-dir -e ".[dev,tracking]"
   ```

6. Copy scenes/ and configs/ directories:
   ```dockerfile
   COPY scenes/ /app/scenes/
   COPY configs/ /app/configs/
   ```

7. Keep existing ENV and ENTRYPOINT/CMD:
   ```dockerfile
   ENV PYTHONPATH=/app/src
   ENV PYTHONUNBUFFERED=1
   ENTRYPOINT ["python", "-m", "surg_rl.cli"]
   CMD ["version"]
   ```

8. Remove ALL `FROM ... as base`/`FROM ... as build`/`FROM ... as runtime` multi-stage blocks and all `COPY --from=build` lines. This eliminates cross-arch binary incompatibility from `COPY --from=build /usr/local/lib/python3.11/site-packages`.
  </action>

  - Dockerfile contains exactly one `FROM` line: `FROM --platform=$BUILDPLATFORM python:3.11-slim`
  - Dockerfile contains `pip install -e ".[dev,tracking]"`
  - Dockerfile does NOT contain `as base`, `as build`, `as runtime`, or `COPY --from=build`
  - Dockerfile ends with `ENTRYPOINT ["python", "-m", "surg_rl.cli"]` and `CMD ["version"]`
  - Line count is 30-40 lines (from current 48)
