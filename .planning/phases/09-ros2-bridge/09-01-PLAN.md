<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| External ROS2 nodes → Ros2BridgeNode subscriber | Untrusted data on `/surg_rl/commands` topic — any ROS2 node on the DDS network can publish |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Spoofing | `Ros2BridgeNode._on_command` | mitigate | Validate message dimension matches `len(self._joint_names)` at subscriber callback entry; mismatch → apply zero action (per D-23) |
| T-09-02 | Tampering | `Ros2BridgeNode._on_command` / `publish_state` | mitigate | Validate no NaN/Inf in qpos/qvel via `np.isfinite()` before publish (per D-25, raises ValueError); validate command data array is numeric |
| T-09-03 | Information Disclosure | `Ros2BridgeNode.publish_state` | accept | JointState data on ROS2 topic is intentional — this is a research tool, not clinical device; DDS discovery publishes topic names by design |
| T-09-04 | Denial of Service | `Ros2BridgeNode._on_command` | mitigate | `queue.Queue(maxsize=1)` with `get_nowait()` keep-latest semantics — fast publishers cannot fill queue; subscriber callback is O(1) |
| T-09-05 | Elevation of Privilege | Bridge process boundary | accept | Bridge runs as separate multiprocessing.Process (per D-01) — uses OS process isolation; no privilege escalation path beyond what process isolation provides |
</threat_model>
