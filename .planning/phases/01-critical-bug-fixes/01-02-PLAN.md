1. Fix BUG-07: In `LightConfig.validate_light_type()` (line 745-760 of schema.py), the method currently does `data["direction"] = (0.0, 0.0, -1.0)` directly on the input `data` dict. In Pydantic v2 with `@model_validator(mode="before")`, mutating `data` in place is risky because Pydantic may copy the model internally and discard the mutation. Fix: use `data = dict(data)` to create a copy at the start of the validator, or use `self.model_copy(update={...})` in an `mode="after"` validator. However, since this is a `mode="before"` validator and it receives a dict, we can safely do `data = dict(data)` at the top to avoid mutating the caller's dict.

Wait, actually in Pydantic v2, `mode="before"` validators receive the raw input dict. It's fine to mutate it in place for the validator's own purposes, but the bug report says "LightConfig validator does not mutate data dict in place". So the fix is to NOT mutate in place: copy the dict before modifying.

Fix:
```python
@model_validator(mode="before")
@classmethod
def validate_light_type(cls, data: Any) -> Any:
    if isinstance(data, dict):
        data = dict(data)  # Copy to avoid mutating input
        light_type = data.get("type")
        ...
    return data
```
