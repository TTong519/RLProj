Current code observations:
- `TrainingManager.evaluate()` (training.py, lines 444–537) uses `is_vec_env = hasattr(eval_env, "num_envs")` to decide API. It handles VecEnv reset (returns obs only) and step (returns obs, reward, done, info). Gymnasium reset returns (obs, info) and step returns (obs, reward, terminated, truncated, info). The current code mostly handles this, but there are two fragilities:
  1. If the VecEnv `reset()` ever returns a tuple (future SB3 versions), `obs = reset_result` will be the tuple, crashing `model.predict`.
  2. VecEnv `info` from `step()` is a list of dicts; the code does `info.get("task_success", False)` which silently drops successes because `info` is a list rather than a dict.
- `Settings.llm_api_key` (config.py, line 75–78) has no validation for placeholder strings.
- `.env.example` (line 8) contains `LLM_API_KEY=your_api_key_here`.
- `utils/logging.py` has no API key masking filter.
</context>

@field_validator("llm_api_key", mode="before")
@classmethod
def reject_placeholder_api_key(cls, v: str | None) -> str | None:
    if v is None:
        return v
    placeholder_patterns = [
        "your_api_key_here",
        "YOUR_API_KEY",
        "REPLACE_ME",
        "sk-xxxxxxxx",
        "xxxxxx",
    ]
    lowered = str(v).lower()
    if any(p.lower() in lowered for p in placeholder_patterns):
        raise ValueError("LLM_API_KEY appears to be a placeholder. Set a real API key.")
    return v

1. SEC-02: In `src/surg_rl/utils/logging.py`, create a `SensitiveDataFilter(logging.Filter)` that masks API keys in log records. The filter should apply to `LogRecord.msg` and `LogRecord.args`. For any string containing a pattern like `sk-[A-Za-z0-9]{20,}` (OpenAI) or `sk-ant-[A-Za-z0-9-]{20,}` (Anthropic), replace each match with `****` + last 4 characters. Example: `sk-abc12345...` becomes `****...d678`.

Implementation:
```python
import re

class SensitiveDataFilter(logging.Filter):
    _API_KEY_RE = re.compile(r"(sk-[A-Za-z0-9]{20,}|sk-ant-[A-Za-z0-9-]{20,})")

    def _mask(self, s: str) -> str:
        def replacer(m: re.Match) -> str:
            full = m.group(1)
            return "****" + full[-4:]
        return self._API_KEY_RE.sub(replacer, s)

    def filter(self, record: logging.LogRecord) -> bool:
        if isinstance(record.msg, str):
            record.msg = self._mask(record.msg)
        if record.args:
            record.args = tuple(self._mask(str(a)) if isinstance(a, str) else a for a in record.args)
        return True
```

2. In `setup_logging`, after creating each handler, add `handler.addFilter(SensitiveDataFilter())`.
