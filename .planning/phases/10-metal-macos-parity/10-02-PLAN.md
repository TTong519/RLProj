    1. Add to `src/surg_rl/utils/gpu.py`:
       ```python
       import warnings

       _mps_fallback_warned: bool = False

       def mps_fallback_to_cpu(operation_name: str) -> None:
           """Log MPS→CPU fallback warning once per training run."""
           global _mps_fallback_warned
           if not _mps_fallback_warned:
               _mps_fallback_warned = True
               logger.warning(
                   "MPS does not support operation '%s' — falling back to CPU. "
                   "This warning appears once per training run.",
                   operation_name,
               )
       ```

    2. This is a utility function. The actual try/except wrapping happens in SB3's internal tensor operations — we cannot inject into SB3's C++/Python layer easily. Instead, provide a **documented pattern**:

       In `src/surg_rl/rl/training.py`, add a comment block near `_create_model()`:
       ```
       # METAL-03: MPS fallback pattern (SB3 handles most ops; for custom code):
       #   try:
       #       result = some_torch_op(tensor.to("mps"))
       #   except RuntimeError as e:
       #       if "MPS" in str(e):
       #           mps_fallback_to_cpu("some_torch_op")
       #           result = some_torch_op(tensor.to("cpu"))
       #       else:
       #           raise
       ```

    3. The primary guarantee is that SB3 + MPS `MlpPolicy` architectures (linear layers, ReLU, tanh) are fully MPS-compatible in PyTorch >=2.3. The fallback exists for custom policy networks or extensions. SB3 `MlpPolicy` with `net_arch=[256,256]` works on MPS natively — no fallback needed for standard training.
