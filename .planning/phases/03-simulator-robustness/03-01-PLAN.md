<objective>
Optimize simulator performance by caching soft-body mesh data between episode resets and vectorizing procedural mesh generation. This reduces reset latency from redundant file I/O and accelerates mesh creation for RL training loops.

Purpose: Surgical RL requires thousands of episode resets per training run. Current implementation regenerates .vtk files on every reset and uses nested Python loops for mesh generation, causing ~500ms+ reset times that bottleneck training throughput.
Output: Cached mesh data in PyBulletSimulator, vectorized NumPy mesh generators, and extended SceneBuilder caching — all verified by timing tests.
</objective>
