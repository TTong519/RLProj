# Phase 02 Plan 03: Gripper + Validation

**Objective:** Implement gripper auto-detection (ACT-04) and load-time action type validation (ACT-05). Ensure gripper dimension is included in action space automatically, and unsupported action types are rejected at scene load.

## Context

- `SurgicalEnv._default_action_config` previously hardcoded `JOINT_POSITIONS` without gripper awareness
- `ActionConfig.include_gripper` defaulted to `False`
- `_control_map` entries include `is_gripper: bool` for end-effector joints
- `scene.robots[].end_effectors` contains gripper names as fallback when control map unavailable
- ACT-05 needs a clear, non-brittle validation hook for future unsupported types

## Tasks

### Task 1: Auto-detect gripper from scene/simulator
- Query `_control_map` for `is_gripper=True` entries after simulator load
- Fallback to `scene.robots[0].end_effectors` list if control map has no gripper markers
- Set `include_gripper = True` and adjust `num_joints = total_controls - gripper_count`
- Ensure `action_type` propagated from config or defaults to `JOINT_POSITIONS`

### Task 2: Add load-time action type validation
- Empty `unsupported_types = ()` allow-list (all ActionTypes implemented)
- Guard: `if action_type in unsupported_types: raise ValueError(...)`
- Non-brittle: future types can be added to the tuple without structural changes

### Task 3: Tests
- **TestGripper**:
  - `test_action_space_includes_gripper_when_configured` (explicit True)
  - `test_step_with_gripper_action_mujoco` (end-to-end MuJoCo)
  - `test_step_with_gripper_action_pybullet` (end-to-end PyBullet)
  - `test_default_action_config_detects_gripper_from_scene` (auto-detection)
- **TestActionTypeValidation**:
  - `test_endeffector_pose_does_not_raise`
  - `test_endeffector_delta_does_not_raise`
