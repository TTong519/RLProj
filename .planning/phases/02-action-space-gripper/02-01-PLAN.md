Current code observations:
- `ActionBuilder.process_action` (action.py line 327-336) explicitly raises `NotImplementedError` for JOINT_TORQUES, ENDEFFECTOR_POSE, and ENDEFFECTOR_DELTA:
  ```python
  if self.config.action_type in (ActionType.JOINT_TORQUES, ActionType.ENDEFFECTOR_POSE, ActionType.ENDEFFECTOR_DELTA):
      raise NotImplementedError(...)
  ```
- MuJoCo `_apply_action` (mujoco_simulator.py line 546-566) currently writes action values directly into `self._data.ctrl`. For torque control, MuJoCo actuators using `motor` type interpret `ctrl` as torque when the actuator is configured for torque (the default). So the change is to skip the POSITION_CONTROL path and write `ctrl` directly, which already happens.
  - HOWEVER, `_build_control_map` in mujoco_simulator.py may only create position controls. We need to verify that when action_type is JOINT_TORQUES, the actuators support torque.
- PyBullet `_apply_action` (pybullet_simulator.py line 1115-1188) currently always uses `POSITION_CONTROL`. For torque control, it must use `TORQUE_CONTROL` mode in `setJointMotorControl2`:
  ```python
  self._pb.setJointMotorControl2(
      body_id, joint_idx, self._pb.TORQUE_CONTROL,
      force=target, physicsClientId=...
  )
  ```
  - Before applying torque, PyBullet must call `setJointMotorControlMultiDofArray` or individual joints with `VELOCITY_CONTROL` or `POSITION_CONTROL` set to zero force, which is a prerequisite for torque control.
- `SurgicalEnvironment._default_action_config` (environment.py line 232-253) defaults to `ActionType.JOINT_POSITIONS` + `num_joints`. The environment must be able to accept a different action type from the scene config.
- Tests in `tests/test_rl_observation_action.py` and `tests/test_rl_environment.py` contain tests for action space building but not for torque mode.
</context>

Alternatively, and more practically: since the robot's URDF or MJCF already defines actuators, and `_apply_action` writes ctrl directly, the simplest fix is to accept that `ctrl` in MuJoCo is the actuator command and different actuator types interpret it differently. 8. Given the complexity of MJCF actuator type changes, the pragmatic approach for this plan is:
