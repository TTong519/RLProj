### Task 5: GAP-05 — Deduplicate Metal detection

**Current state:** `config.py:18-25` has standalone `_mps_available()`:
```python
def _mps_available() -> bool:
    try:
        import torch
        return bool(torch.backends.mps.is_available())
    except Exception:
        return False
```
`gpu.py:88-104` has `_has_metal()` with broader detection (system_profiler + torch fallback).

**Action:**
- Replace `_mps_available()` body with call to `gpu._has_metal()` or a new public `is_metal_available()` exported from `gpu.py`
- Since `_has_metal()` returns `bool`, and `_mps_available()` returns `bool`, the call site `config.py:23` (`return bool(torch.backends.mps.is_available())`) can be replaced with:
  ```python
  from surg_rl.utils.gpu import _has_metal
  
  def _mps_available() -> bool:
      return _has_metal()
  ```
- Alternative: replace all callers of `_mps_available()` with direct import and remove the function entirely

**Verify:** `_has_metal()` is imported; `_mps_available()` delegates to it. No duplicated torch import logic remains in config.py.
