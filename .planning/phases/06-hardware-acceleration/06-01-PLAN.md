       - `def select_backend(requested: HardwareBackend) -> HardwareBackend`: If requested is auto, pick `detect_backends()[0]`. If requested is cpu, return cpu. If requested is in `detect_backends()`, return it. Otherwise log INFO: "Requested backend {req} not available. Available: {list}. Falling back to cpu." and return cpu. In all non-error paths, log INFO: "Selected backend: {backend.value}".
       - `def get_cuda_version() -> str | None`: runs nvidia-smi for driver_version, returns first line or None.
       - `def get_rocm_version() -> str | None`: runs `rocminfo | grep "ROCm version"`, returns version string or None.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| subprocess.run() → system binaries | Untrusted PATH could inject fake nvidia-smi/rocminfo |
| INFO logs → console / log files | GPU capabilities exposed in logs (low sensitivity) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Tampering | gpu.py detection | mitigate | Use `shutil.which()` on known binary names; subprocess timeout=5 prevents hanging on malicious binaries |
| T-06-02 | Information Disclosure | INFO logs | accept | GPU availability is not sensitive PII; research tool context makes this acceptable |
| T-06-03 | Denial of Service | subprocess.run() | mitigate | All subprocess calls have 5-second timeout; exceptions caught and return False (graceful degradation) |
| T-06-04 | Elevation of Privilege | rocminfo /dev/kfd | mitigate | Permission error detected explicitly; logged as INFO (not ERROR) with remediation guidance |
</threat_model>
