## Integration Audit

### Phase 15 → 16 (Tetgen → MuJoCo flex) — YELLOW → GREEN
- **Original finding:** No bridge from in-memory tetgen numpy arrays to on-disk .node/.ele files
- **Fix:** `_add_tissue_to_mjcf` now calls `_try_external_tetrahedralization` in-memory and passes (vertices, elements) directly to `_add_flex_body_to_mjcf`. `_add_flex_body_to_mjcf` accepts `vertices`/`elements` numpy array params, falling back to file I/O only when arrays not provided.
- **File:** `src/surg_rl/simulators/scene_builder.py:785-798`

### Phase 15 → 17 (Tetgen → Cutting) — YELLOW → GREEN
- **Original finding:** PyBullet `_apply_cut` uses `self._scene_definition` which is never assigned (AttributeError)
- **Fix:** Changed `self._scene_definition` to `self._scene` at `pybullet_simulator.py:1268`
- **File:** `src/surg_rl/simulators/pybullet_simulator.py:1268`

### Phase 18 → Environment (Fluids → env) — RED → GREEN
- **Original finding:** FluidSimulator never instantiated; FluidConfig is an orphan field
- **Fix:** `_init_fluid()` added to `SurgicalEnv.__init__`, instantiating `FluidSimulator` when `scene.fluid.enabled`. Fluid step hook in `step()` runs at `substep_dt` rhythm via `self._step_count % interval`.
- **File:** `src/surg_rl/rl/environment.py:218-219` (init), `676-687` (method), `622-626` (step hook)

### Dependencies — YELLOW → GREEN
- **Original finding:** phiflow not in pyproject.toml
- **Fix:** Added `[simulation]` extras: `phiflow>=3.4.0`, `scikit-image>=0.21.0`
- **File:** `pyproject.toml:85-88`
