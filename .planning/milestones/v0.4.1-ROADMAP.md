## Milestone Summary

**Key Decisions:**

- **D-25-01**: `SurgicalEnv.passthrough_step()` is the cleanest solution for MARL per-arm action passthrough (vs. allowing zero-sized action arrays in `step()`). Avoids 3-line `_get_action_size()` branching logic in the env step.
- **D-25-02**: Extract `_step_simulator_and_build_outputs(processed_action, source_action)` helper so `step()` and `passthrough_step()` share ~90 lines of post-step observation/reward building. Eliminates drift risk.
- **D-26-02**: `_JsonStdout` wrapper class (with `write()` and `flush()`) replaces `os.fdopen` on PyTorch's non-blocking Pipe. Retains `os.fdopen` for real stderr (FD 2).
- **D-27-06**: 5 new task scene JSONs aligned with Phase 24 `test_dreamer_training.py` parametrize contract (instrument + tissue types per task) â€” discovered during task 2 that initial scene choices (e.g. `needle_driver` for needle_insertion) violated existing test expectations.
- **D-27-09**: `ExperimentRunner.__init__` writes `experiments/{name}.yaml` to fix misleading CLI hint (`Reproduce with: --config experiments/{name}.yaml` previously pointed at a non-existent file).
- **D-28-01**: Retroactive VERIFICATION.md files use `retroactive: true` and `retro_audit:` frontmatter fields to distinguish from forward-phase VERIFICATION files.
- **D-28-02**: Phase 21 SC-2 (TASK-02) marked `passed-partial` â€” router activation closed by Phase 27, 3-difficulty-levels portion documented as Residual Gap (v0.5.0 backlog owner).
- **D-28-03**: Only BENCH-01 body checkbox flipped to `[x]` in REQUIREMENTS.md; TASK-02 left at `[ ]` (3-difficulty-levels still open); BENCH-02..05 left at `[ ]` (out of Phase 28 scope, acknowledged as low-severity process gap).
