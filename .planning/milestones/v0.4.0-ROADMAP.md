# Milestone v0.4.0: Training Infrastructure & Realism

**Status:** ✅ SHIPPED 2026-06-09
**Phases:** 19-24
**Total Plans:** 21 (19: 3 plans, 20: 4 plans, 21: 3 plans, 22: 3 plans, 23: 3 plans, 24: 5 plans incl. gap closure)

## Overview

Transform Surg-RL from a simulation framework into a competitive RL research platform with real surgical assets, comprehensive task curriculum, systematic benchmarking, multi-agent support, and DreamerV3 world models. v0.4.0 ships a schema-first foundation (Pydantic v2 + optional dependency groups), real instrument and organ meshes via trimesh, a six-task curriculum with progressive difficulty, dual-arm PettingZoo multi-agent training, an SB3 benchmarking framework with publication-quality reporting, and a feasibility-driven DreamerV3 world model integration with process isolation.

## Milestone Summary

**Key Decisions:**

- **Schema-first**: All new Pydantic v2 models with `None` defaults; existing models unchanged. Decouples schema from implementation.
- **Optional dependency groups**: `[assets]`, `[benchmark]`, `[marl]`, `[dreamer]` declared in pyproject.toml. `import surg_rl` works without any of them. Lazy imports via shared `LazyImport` helper.
- **trimesh is sole new mesh library**: no glTF/COLLADA complexity; OBJ is universal baseline.
- **PettingZoo ParallelEnv as separate class from SurgicalEnv**: clean adapter pattern, no shared mutable state.
- **Dual statistical aggregation (mean±1σ + IQM+CI)**: per D-08, both methods shown for transparency in benchmark reports.
- **Seaborn colorblind-safe palette with fixed algorithm color cycle**: deterministic plot colors across runs.
- **DreamerV3 process isolation via JAX subprocess**: `XLA_PYTHON_CLIENT_MEM_FRACTION=0.4` prevents JAX+PyTorch GPU memory conflict; stdin/stdout JSON-line protocol with ACK handshakes.
- **GymToEmbodiedWrapper with reset-in-action protocol**: reset signal embedded in action dict, observations as flat dicts with is_first/is_last/is_terminal.
- **DreamerV3 feasibility spike with kill switch**: DMV3-01 determines if RSSM can model tet mesh cutting dynamics; if it fails, full integration deferred to v0.5.0 per DMV3-05.
- **Benchmarking treats MuJoCo and PyBullet as separate targets**: no cross-backend aggregation; per-backend reporting only.
- **CurriculumScheduler extension is additive**: never replaces Phase 3 fix; all task curriculum additions are extension-only.
- **DreamerConfig obs_type supports both pixel and low-dim state**: 64×64 RGBA pixel render or ~50-100 dim state vector.
- **Auto-discovery of DreamerV3 checkpoints**: `models/dreamerv3/{task}_{obs_type}/` pattern; benchmark can include them with `--dreamer-comparison` flag.

**Issues Resolved:**

- Phase 3 CurriculumScheduler `apply_parameters` integration with TaskResult: pure extension, original behavior preserved
- MuJoCo `<flexcomp>` vs `<flex>` choice for arbitrary tetgen meshes: `<flex>` chosen for native tet mesh support
- V-HACD convex decomposition for instrument collision: avoids concave mesh artifacts in PyBullet
- rliable library for IQM/stratified bootstrap CI: installed via `[benchmark]` extra
- DreamerV3 + PyTorch GPU memory conflict: resolved via subprocess isolation with XLA memory fraction
- Task curriculum difficulty parameter interpolation: `interpolate_params()` smooth transition between easy/medium/hard
- Pydantic v2 model_dump Enum handling: convert to `.value` strings before YAML serialization (avoids RepresenterError)
- DreamerV3 benchmark integration conditional on spike: "pending — Phase 24" status shown when no checkpoints
