## v1 Requirements — All Shipped

### Phase 15: Tetgen Mesh Generation

- [x] **TETG-01**: Integrate `tetgen` Python package as the primary tetrahedral mesh generator, replacing VTK-based fallback in `vtk_io.py`
  - **Outcome:** ✅ Implemented. `_try_external_tetrahedralization` in `mesh_generation.py` calls `tetgen.TetGen` for box, sphere, cylinder generators. VTK I/O preserved for file format compatibility.
- [x] **TETG-02**: Generate tetrahedral meshes from surface OBJ/STL inputs via tetgen CLI or Python bindings
  - **Outcome:** ✅ Implemented. OBJ → surface triangulation → tetgen.TetGen pipeline functional.
- [x] **TETG-03**: Remove VTK dependency from project `[meshing]` extras; tetgen becomes platform-agnostic
  - **Outcome:** ✅ Implemented. pyproject.toml `[meshing]` extras: `tetgen>=0.8.4` only (no pyvista).
- [x] **TETG-04**: Preserve existing `vtk_io.py` public API but redirect internals to tetgen
  - **Outcome:** ✅ Implemented. All public functions (write/read/validate) unchanged. 21 tests pass.

### Phase 16: Deformable Objects

- [x] **DEFM-01**: Add FEM-based deformable objects in MuJoCo via `<deformable>/<flex>` elements in MJCF/XML
  - **Outcome:** ✅ Implemented. `_add_flex_body_to_mjcf` generates flex from tetgen meshes. In-memory and file-based paths both supported.
- [x] **DEFM-02**: Improve PyBullet soft body parameter mapping from `TissueConfig.physics.pybullet` to `loadSoftBody`
  - **Outcome:** ✅ Implemented. Neo-Hookean μ/λ auto-derived from E,ν. DeformableConfig.pybullet routing integrated.
- [x] **DEFM-03**: Unified `DeformableConfig` in scene schema with backend-specific overrides
  - **Outcome:** ✅ Implemented. MuJoCoFlexConfig, PyBulletFlexConfig, BoundaryCondition models. No cross-contamination.
- [x] **DEFM-04**: Deformable state observable via `ObservationConfig` (vertex positions, strain)
  - **Outcome:** ✅ Implemented. Dynamic `build_deformable_spec(max_vertices)`, padded vertex extraction, per-edge strain.

### Phase 17: Volumetric Cutting

- [x] **CUT-01**: Real-time tetrahedral mesh cutting — detect intersection between cutting tool and tetrahedral mesh
  - **Outcome:** ✅ Implemented. `compute_signed_distances`, `edge_intersection`, `classify_tet_case` (5 canonical cases).
- [x] **CUT-02**: Remesh cut faces — split tets along cut plane, generate new boundary faces
  - **Outcome:** ✅ Implemented. `cut_tetrahedral_mesh` with 3-1/2-2 subdivision, vertex dedup, cut face extraction.
- [x] **CUT-03**: Integrate with MuJoCo and PyBullet simulation
  - **Outcome:** ✅ Implemented. MuJoCo: MJCF XML inline rewrite + model reload. PyBullet: safe reload with stored tetrahedra.
- [x] **CUT-04**: `CutAction` in action space schema
  - **Outcome:** ✅ Implemented. Pydantic v2 model with direction normalization, discrete trigger, 25-step cooldown.

### Phase 18: Grid-based Fluids

- [x] **FLUD-01**: Eulerian grid-based fluid solver — velocity/pressure fields on staggered grid, MAC method
  - **Outcome:** ✅ Implemented. PhiFlow 3.4.0 StaggeredGrid, MacCormack advection, pressure projection (sparse CG).
- [x] **FLUD-02**: Two-way fluid-solid coupling — fluid forces on objects, object motion affecting fluid
  - **Outcome:** ✅ Implemented. Pressure gradient integration for force computation. Multi-obstacle union() workaround.
- [x] **FLUD-03**: Scene schema extension — `FluidConfig` with domain bounds, resolution, viscosity, density
  - **Outcome:** ✅ Implemented. FluidConfig + FluidBoundaryType in schema.py. `fluid` field on SceneDefinition.
- [x] **FLUD-04**: Fluid rendering — simple particle or surface visualization for debugging and demo
  - **Outcome:** ✅ Implemented. 2D colormesh via skimage. render_fluid_2d() public API. 2 visualization tests.
