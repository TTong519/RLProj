# Requirements: Surg-RL v0.5.0 Scene Editor & UX Polish

**Defined:** 2026-06-18
**Core Value:** End-to-end pipeline from a text description or JSON scene definition to a trained RL policy in a realistic surgical simulation — with automatic primitive fallbacks when real assets are missing, and a benchmarking framework for systematic RL research comparisons.

**Milestone scope:** 4 user-confirmed categories — PySide6 GUI scene editor (5 sub-components), demo suite polish (3 demos), user-facing docs refresh, tech debt cleanup (6 items). Marquée feature is the GUI editor; demos and docs support adoption.

---

## v1 Requirements

### GUI — PySide6 Scene Editor

PySide6 6.8+ (LGPL-3) ships as a new `[gui]` optional extra (~120 MB off the headless install path). Editor lives in a leaf-consumer `src/surg_rl/editor/` package and exposes a `surg-rl-gui` console script (NOT a Typer subcommand) to preserve the 14-subcommand backwards compat. The editor embeds the existing MuJoCo/PyBullet simulators via a thin `SimRenderBridge` (no PyOpenGL/VisPy/pyqtgraph). A pure-Python `SchemaWalker` reads `SceneDefinition.model_json_schema()` and renders type-aware Qt widgets via a `FieldRenderer` registry (no if/elif chain rot).

- [ ] **GUI-01**: User can launch the scene editor via `surg-rl-gui [scene.json]` console script (registered in pyproject.toml `[project.scripts]`); without `[gui]` installed, the script prints install hint and exits non-zero
- [ ] **GUI-02**: User can open / save `.planning/scenes/*.json` files (existing schema), with auto-validation on save (Pydantic v2 round-trip preserves all 63 schema classes including the `_FloatMixin` `DifficultyLevel` enum)
- [ ] **GUI-03**: User sees a 3D viewport of the loaded scene (orbit/pan/zoom via mouse, camera reset keybinding, 20 Hz FPS throttle) backed by `sim.render_to_numpy(w, h) -> np.ndarray` on both MuJoCo and PyBullet simulators
- [ ] **GUI-04**: User sees a tree view of the scene elements (instruments, organs, tissues, lighting, curriculum, task config); right-click context menu supports add/remove/duplicate; drag-reorder within parents; validation icons (red dot for invalid, green check for valid)
- [ ] **GUI-05**: User sees a property form editor for the selected tree node; fields are auto-generated from `SceneDefinition.model_json_schema()` via a `SchemaWalker` + `FieldRenderer` registry; widgets are type-aware (vec3 spinbox, enum combobox, file path picker, color picker, range slider); validation errors appear inline below the field
- [ ] **GUI-06**: User can undo / redo any property change (Cmd+Z / Cmd+Shift+Z), with stack scoped per scene; undo persists for the session, not across saves (deferred: cross-save undo)
- [ ] **GUI-07**: User sees an LLM-prompt-to-JSON panel: text input box → background QThread worker calls existing `TextParser.parse_sync()` → preview pane shows generated JSON → user accepts (writes to draft scene) or rejects (clears preview); user can regenerate with modified prompt
- [ ] **GUI-08**: Editor shell provides standard workspace layout (viewport center, tree left, properties right, LLM panel bottom); file menu (New / Open / Save / Save As / Recent Files); drag-drop `.json` onto window opens it; CLI integration: `surg-rl-gui path/to/scene.json` opens that scene directly
- [ ] **GUI-09**: All LLM API keys and error messages pass through a regex redactor (`safe_error_message()`) before reaching logs or UI; LLM panel settings use `QSettings` with platform-appropriate storage; provider switching is live (no restart)
- [ ] **GUI-10**: Editor is gated by `[gui]` optional dep group with a `LazyImport` + `HAS_GUI` sentinel following the proven 7-place pattern (mirrors `[marl]`, `[dreamer]`, `[vision]`, `[benchmark]`); `surg-rl train` and other 13 subcommands do not import PySide6 even when `[gui]` is installed

### DEMO — Demo Suite Polish

Three demos share a common narration script (`demos/_common.py` with banner, scene resolver, regression-test contract). Each demo runs `--headless --steps 0` for the CI smoke test and `--headless --steps N` for the user-facing walkthrough. Existing suturing demo (1168-test clean from quick task 20260617-demo-rework) is refactored to use `_common.py`; knot-tying and needle-passing demos are new.

- [ ] **DEMO-01**: User can run `python demos/suturing_demo.py --headless` and see a consistent Rich banner + scene info + narrated step output + eval printout, matching the 2026-06-17 suturing demo style
- [ ] **DEMO-02**: User can run `python demos/knot_tying_demo.py --headless` and see the same consistent shell; scene loaded from `tests/fixtures/scenes/knot_tying.json` (existing v0.4.0 scene); 3-4 step narration covering needle insertion, knot formation, knot tightening
- [ ] **DEMO-03**: User can run `python demos/needle_passing_demo.py --headless` and see the same consistent shell; scene loaded from existing needle_passing JSON; 3-4 step narration covering needle pick-up, target approach, pass-through, withdrawal
- [ ] **DEMO-04**: Each demo has a regression test in `tests/test_demos.py` (subprocess-based: each demo runs `--headless --steps 0`, asserts exit 0 and expected banner substring); CI runs all 3 demo regression tests on Linux + macOS
- [ ] **DEMO-05**: A `demos/NARRATION_TEMPLATE.md` documents the shared narration structure (banner / scene info / per-step output / eval) so future demos follow the same pattern; this template is written FIRST per pitfall P8 prevention

## Out of Scope

Carried forward from v0.4.2 archive plus v0.5.0-specific exclusions:

| Feature | Reason |
|---------|--------|
| Mobile app | Web/library-first; mobile is a different product |
| Real-time multi-user networked surgery | Single-agent / dual-agent training scope |
| FDA certification / medical-grade safety validation | Research / simulation tool, not clinical device |
| Unity/Unreal rendering backends | MuJoCo + PyBullet rendering sufficient |
| DirectML / Vulkan compute backends | Windows not primary target |
| Linux-only ROS2 subscriber e2e tests | Requires real ROS2 runtime; mock coverage sufficient |
| Helm chart | Kustomize overlays sufficient |
| Real-time ROS2 DDS router for K8s multicast | DDS multicast is platform-level; document workaround |
| 3D fluid simulation | 2D xz-plane slice sufficient; 3D behind `dim_3d=True` flag |
| GPU fluid acceleration | PhiFlow CPU-first |
| COLLADA/glTF mesh format | OBJ universal baseline |
| TASK-02 per-level override schema (`DifficultyLevelConfig`) | D-29-03 explicit exclusion; deferred to v0.6.0+ |
| TASK-02 `CurriculumScheduler` discrete level progression | D-29-03 explicit exclusion; deferred to v0.6.0+ |
| TASK-02 scene-level `difficulty_levels: list[3]` blocks | D-29-03 explicit exclusion; deferred to v0.6.0+ |
| Real dreamerv3 integration (flips Phase 30 stub-state sentinel) | Out of v0.5.0 scope; deferred to v0.6.0+ when GPU infrastructure is ready |
| K8s PVC e2e tests | Requires PVC provisioner; deferred to v0.6.0+ |
| KubeRay prerequisite tests | Deferred to v0.6.0+ |
| Organ mesh source licensing (surgtoolloc, MakeHuman, BodyParts3D) | Requires legal review; research-only spike in Phase 35 |
| Per-tet generation counter for degenerate tets | Single cut per episode typical; deferred indefinitely |
| New task types beyond the existing 6 | TASK-01..04 locked at 6 types in v0.4.0 |
| Deep API tutorials (mkdocs + mkdocstrings) | User-facing docs only in v0.5.0; deferred to v0.6.0+ |
| Schema walker JSON-Schema-driven form (vs. Pydantic introspection) | Direct introspection more accurate; JSON-Schema reserved for debug panel; deferred |
| Editor auto-save (vs. explicit Cmd+S) | Explicit save by default; auto-save is per-user preference toggle, deferred |
| PyQt6 instead of PySide6 | PyQt6 is GPL-3, would force Surg-RL to GPL; PySide6 is LGPL-3 compatible |
| Tauri / Electron / Tkinter | PySide6 selected per user direction; Tauri heavier install, Tkinter limited 3D |
| PyOpenGL / VisPy / pyqtgraph in viewport | MuJoCo `Renderer` + PyBullet `getCameraImage` → QImage is sufficient and Qt-context-clean |
