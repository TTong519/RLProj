## v1 Requirements (Archived)

### Real Assets (all complete)

- ✓ **ASET-01**: System loads OBJ instrument meshes (forceps, scalpel, needle driver, retractor, trocar, scissors, clamp, plus 3 general-purpose instruments) via trimesh, generating URDF/MJCF collision geometry for both MuJoCo and PyBullet backends — **v0.4.0, validated Phase 20**
- ✓ **ASET-02**: System loads 4 organ meshes (liver, stomach, kidney, gallbladder) as OBJ surface meshes, converting them through the existing tetgen pipeline for deformable simulation with volumetric cutting support — **v0.4.0, validated Phase 20**
- ✓ **ASET-03**: All real meshes silently fall back to existing primitive geometry when the mesh file is not found at the configured path — no crashes, no breaking changes to existing scenes — **v0.4.0, validated Phase 20**
- ✓ **ASET-04**: Mesh assets support configurable decimation (target face count) to control simulation performance without manual mesh editing — **v0.4.0, validated Phase 20**

## Out of Scope (Carried Forward, Unchanged)

| Feature | Reason |
|---------|--------|
| Real-time multi-user networked surgery | Single-agent / dual-agent training scope |
| FDA certification / medical-grade validation | Research and simulation tool, not clinical device |
| Task chains (grasp→cut→suture) | Deferred to v0.5.0 — composite scheduling requires novel infrastructure |
| RLlib MARL centralized critic | Independent SB3 policies via SuperSuit sufficient for dual-arm |
| 3D DreamerV3 video prediction | 2D pixel reconstruction sufficient for feasibility assessment |
| COLLADA/glTF mesh format support | OBJ is universal baseline; multi-format adds complexity |
| Helm chart | Kustomize overlays sufficient; Helm can be added later |
