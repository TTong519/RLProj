# Milestone v0.3.2: Advanced Simulation

**Status:** ✅ SHIPPED 2026-05-05
**Phases:** 15-18
**Total Plans:** 9 (4 executed, 5 inlined)

## Overview

Advanced simulation features for the surgical robotics RL training system: platform-agnostic tetgen mesh generation, FEM-based deformable objects, real-time volumetric tetrahedral mesh cutting, and Eulerian grid-based fluid simulation with two-way solid coupling.

## Phases

### Phase 15: Tetgen Mesh Generation

**Goal:** Replace VTK-based tetrahedral mesh generation with platform-agnostic `tetgen` Python package.

**Depends on:** None
**Plans:** 1 plan (1 wave)

Plans:
- [x] 15-01-PLAN.md — Integrate tetgen, migrate vtk_io.py internals, update deps (TETG-01..04)

**Details:**
- Replaced PyVista `delaunay_3d()` with `tetgen 0.8.4` for all mesh generators (box, sphere, cylinder)
- `vtk_io.py` public API (write_vtk_unstructured_grid, read_vtk_unstructured_grid, validate_vtk) preserved
- VTK/PyVista removed from `[meshing]` extras (200MB dep savings)
- 21 tests (tetgen integration + mesh generation + vtk_io + performance)

### Phase 16: Deformable Objects

**Goal:** FEM-based deformable objects in MuJoCo + improved PyBullet soft body support.

**Depends on:** Phase 15
**Plans:** 2 plans (2 waves)

Plans:
- [x] 16-01-PLAN.md — MuJoCo FEM `<flex>` generation + DeformableConfig schema (DEFM-01, DEFM-03)
- [x] 16-02-PLAN.md — PyBullet soft body param mapping + deformable observation (DEFM-02, DEFM-04)

**Details:**
- DeformableConfig, MuJoCoFlexConfig, PyBulletFlexConfig, BoundaryCondition Pydantic v2 models
- MuJoCo `<deformable>/<flex>` MJCF generation from tetgen node/ele files with in-memory fallback
- Backward-compatible `<flexcomp>` fallback for non-tetgen tissues
- Neo-Hookean μ/λ auto-derived from Young's modulus + Poisson's ratio
- Dynamic TISSUE_DEFORMATION_SPEC with configurable max_vertices (default 200)
- Per-edge strain computation and padded vertex observation extraction
- 31 tests (9 schema + 7 MJCF generation + 8 PyBullet params + 8 observation)

### Phase 17: Volumetric Cutting

**Goal:** Real-time tetrahedral mesh cutting with remeshing, integrated with both simulator backends.

**Depends on:** Phase 15 (tetgen), Phase 16 (deformables)
**Plans:** 3 plans

Plans:
- [x] 17-01-PLAN.md — Cutting algorithm (intersection, remeshing)
- [x] 17-02-PLAN.md — Backend integration (MuJoCo + PyBullet)
- [x] 17-03-PLAN.md — Action space + scene schema

**Details:**
- Pure NumPy tetrahedral mesh cutting engine in `src/surg_rl/cutting/`
- Signed distance classification, edge-plane intersection, 5 canonical tet-plane cases
- 3-1 and 2-2 subdivision with vertex deduplication via hash-based `_get_or_create_intersection`
- MuJoCo integration: model reload via MJCF XML inline vertex/element rewrite + qpos/qvel restore
- PyBullet integration: safe reload pattern (RESET_USE_DEFORMABLE_WORLD + full scene reload) with stored tetrahedra
- CutAction Pydantic v2 schema with mode=before direction normalization
- trigger_cut() on SurgicalEnv with 25-step cooldown (~500ms)
- 17 tests (7 intersection + 5 engine + 3 schema + 1 MuJoCo + 1 PyBullet)

### Phase 18: Grid-based Fluids

**Goal:** Eulerian grid-based fluid solver with two-way solid coupling for surgical bleeding/irrigation.

**Depends on:** None (standalone subsystem)
**Plans:** 3 plans (inlined)

Plans:
- [x] 18-01-PLAN.md (inlined) — Fluid solver core (MAC grid, pressure projection)
- [x] 18-02-PLAN.md (inlined) — Two-way coupling + scene integration
- [x] 18-03-PLAN.md (inlined) — Schema + visualization

## Milestone Summary

**Key Decisions:**
- Tetgen replaces VTK entirely (not side-by-side) — removes PyVista/VTK binary deps
- MuJoCo `<flex>` (not `<flexcomp>`) for arbitrary tetgen meshes
- Cutting is a discrete trigger (not continuous action) with 500ms cooldown
- PyBullet cuts use RESET_USE_DEFORMABLE_WORLD + full reload (removeBody() unsafe)
- PhiFlow over Mantaflow for Eulerian fluids (Mantaflow dead since 2022)
- CPU-first fluids (GPU deferred)
- In-memory tetgen → MJCF flex bridge added during milestone audit

**Issues Resolved:**
- Critical: PyBullet `_apply_cut` AttributeError (self._scene_definition → self._scene)
- Phase 15→16 MuJoCo bridge: in-memory tetgen numpy arrays now flow to _add_flex_body_to_mjcf
- Phase 18→env: FluidSimulator init + step hook added to SurgicalEnv
- Phase 17-02 regression: restored accidentally-deleted set_body_pose/get_body_pose
- Phase 16-02 pre-existing: test_rl_observation_action tissue_deformation shape 50→200
- Indentation bug: SceneDefinition methods duplicated into FluidConfig (fixed)

**Issues Deferred:**
- Per-tet generation counter for degenerate tets after multiple cuts (Phase 17)
- Cut cooldown unit test (requires full env lifecycle)
- Fluid step hook in base_simulator.py (env-level hook sufficient)

**Technical Debt Incurred:**
- PhiFlow PhiML tensor → numpy extraction quirks on Python 3.13+ (workarounds in force computation / viz)
- PhiFlow multi-obstacle union() bug requires merged SDF workaround (documented pitfall)
- 2D fluid only (xz-plane slice); 3D behind dim_3d=True flag, not yet implemented
