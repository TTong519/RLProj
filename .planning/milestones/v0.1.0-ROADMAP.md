# Milestone: v0.1.0 — Stabilization

**Status:** Complete

## Summary

This milestone stabilizes the Surg-RL surgical-robotics RL training system (v0.1.0) by fixing 8 critical bugs, completing the action space, hardening simulator performance, and extending task geometry and real asset support. It is a correctness-first, stabilization-focused plan: no new features until the foundation is solid.

## Key Accomplishments

1. **Critical bugs fixed (8/8)** — Quaternion order, joint reset leakage, physics=None crash, reward sign contract, curriculum dynamics overrides, LightConfig mutation, API key exposure, VecEnv evaluate() robustness
2. **Action space complete** — All 7 ActionTypes implemented (JOINT_POSITIONS, JOINT_VELOCITIES, JOINT_TORQUES, ENDEFFECTOR_POSE, ENDEFFECTOR_DELTA, DISCRETE, GRIPPER), gripper auto-detection in both backends
3. **Simulator performance hardened** — Soft-body mesh caching (<100ms reset), vectorized mesh generation (~10x faster), cross-backend state save/restore (qpos/qvel within 1e-6), persistent eval env caching
4. **Task geometry + real assets** — `target_body` observation binding, real URDF/OBJ loading with deduplicated fallback warnings, MuJoCo MJCF mesh assets
5. **Experiment tracking + CI/CD** — Optional W&B/MLflow callbacks, multi-stage Dockerfile, GitHub Actions CI (matrix 3.10/3.11/3.12) + PyPI release pipeline
