# Requirements: Surg-RL v0.3.0

**Defined:** 2026-05-03
**Milestone:** v0.3.0 — Production & Cross-Platform
**Core Value:** End-to-end pipeline from a text description or JSON scene definition to a trained RL policy in a realistic surgical simulation

## v1 Requirements

Requirements for v0.3.0 release. Each maps to roadmap phases.

### Metal GPU Compute

- [ ] **METAL-01**: `TrainingConfig.device="auto"` resolves to `"mps"` on Apple Silicon when `torch.backends.mps.is_available()`, wiring SB3 training to MPS backend
- [ ] **METAL-02**: MPS device selection is logged at INFO level including unified memory info and a note that MPS is 2-4x slower than CUDA
- [ ] **METAL-03**: Unsupported MPS operations fall back to CPU with a single WARNING log (not per-op spam); training continues without crashing

### macOS Test Parity

- [ ] **MACOS-01**: GitHub Actions CI includes `macos-latest` in the test matrix alongside `ubuntu-latest`
- [ ] **MACOS-02**: All 6 PyBullet soft-body `@pytest.mark.xfail` markers (currently XPASS on macOS) are removed from `tests/test_simulators.py`
- [ ] **MACOS-03**: Viewer/rendering tests on macOS CI runner execute under `mjpython` without manual intervention
- [ ] **MACOS-04**: Linux-only ROS2 subscriber e2e markers remain with documented rationale in Out of Scope

### Multi-platform Docker

- [ ] **DOCKR-01**: CPU Dockerfile supports `linux/amd64` + `linux/arm64` multi-arch builds via `docker buildx`
- [ ] **DOCKR-02**: CUDA Dockerfile publishes amd64 variant; arm64 Jetson variant exists as separate `Dockerfile.jetson`
- [ ] **DOCKR-03**: Release workflow pushes multi-arch container images to GHCR on `v*` git tags
- [ ] **DOCKR-04**: CI workflow includes `docker buildx` cross-architecture build verification with QEMU emulation

### ros2_control Integration

- [ ] **R2CTL-01**: Python `hardware_interface.SystemInterface` subclass wraps `BaseSimulator` state reads (position, velocity) and command writes, registering with `controller_manager`
- [ ] **R2CTL-02**: URDFs generated by `scene_builder` include `<ros2_control>` tags describing joint command/state interfaces
- [ ] **R2CTL-03**: Bridge lifecycle (`_setup_bridge`/`_teardown_bridge`) manages controller manager service startup and controller spawning/stopping
- [ ] **R2CTL-04**: `surg-rl ros2-control` CLI command starts the bridge with ros2_control, exposing configurable controller YAML path

### ROS2 Launch Files

- [ ] **LAUNCH-01**: `.launch.py` files compose bridge node, trajectory replay node, and simulator in a `LaunchDescription`
- [ ] **LAUNCH-02**: Launch files ship in the pip package; documented `ROS_PACKAGE_PATH` fallback supports both colcon workspace and pip-only workflows
- [ ] **LAUNCH-03**: Launch arguments expose scene path, model checkpoint path, and ROS2 topic names as configurable `DeclareLaunchArgument` entries

### Kubernetes Deployment

- [ ] **K8S-01**: K8s Job manifest runs RL training with GPU node selectors and resource requests/limits
- [ ] **K8S-02**: KubeRay `RayCluster` config supports RLlib distributed training with `ray.init(address="auto")` (respecting `RAY_ADDRESS` env var)
- [ ] **K8S-03**: ROS2 bridge runs as sidecar container in the training pod, sharing network namespace
- [ ] **K8S-04**: K8s ConfigMap injects scene definition JSON; K8s Secrets inject API keys and model registry tokens
- [ ] **K8S-05**: PVC-backed checkpoint directory survives pod restarts; TensorBoard logs persist to the same volume

## Out of Scope

| Feature | Reason |
|---------|--------|
| Linux-only ROS2 subscriber e2e tests | Requires real rclpy runtime; macOS mock coverage sufficient (MACOS-04) |
| Helm chart | Kustomize overlays sufficient for v0.3.0; Helm can be added later |
| Windows CI runner | Windows not a primary target; DirectML/Vulkan already excluded |
| Real-time ROS2 DDS router for K8s multicast | DDS multicast issue is platform-level; document workaround, don't solve |
| MoveIt integration | Beyond scope of ros2_control; raw command/state interfaces are sufficient |
| MPS performance parity with CUDA | Physical hardware limitation; documented expectation (METAL-02) |
