# Architecture Decision Records

This directory holds the surg-rl project's Architecture Decision Records
(ADRs). An ADR records a significant decision and its rationale so that
future work has a single, auditable source of truth: any reader can later
understand *what* was decided, *why*, and *what alternatives were rejected*.
ADR-0001 below is the repository's first ADR and sets the template for
future entries.

ADRs in this repository follow the [MADR (Markdown Any Decision
Record)](https://adr.github.io/madr/) format: each record carries a
metadata block (Date, Status, Deciders, Tags) and the sections
`Context`, `Considered Options`, `Decision Outcome`, `Rationale`,
`Consequences`, and `References`. New ADRs should be numbered with a
4-digit zero-padded prefix (e.g. `0002-...`) and added as a new row in
the table below.
